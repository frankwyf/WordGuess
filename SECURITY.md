# Security Policy

## Supported Versions

Only the latest release line is supported with security fixes.

## Reporting a Vulnerability

If you discover a security issue, please do not open a public issue immediately.

1. Open a private vulnerability report through GitHub Security Advisories (preferred).
2. Include reproduction steps and impact details.
3. Allow time for investigation and patching before public disclosure.

## Scope

Security reports should focus on vulnerabilities in this repository and its release artifacts.
